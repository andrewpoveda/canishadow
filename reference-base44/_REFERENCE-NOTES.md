# reference-base44 — read-only reference, DO NOT build here

This folder is the **exported base44 MVP** of CanIShadow (shadow-path-map). It is a
Vite + React app generated by base44 during the base44 × Tavily hackathon (5th place).

It is here purely as **reference** for the clean rebuild on our own stack
(Next.js App Router + Supabase). Do not run, deploy, or extend this code — read it,
port the good parts, and leave it untouched.

## What to reuse (the valuable, non-base44-locked parts)

- **Data model** — `base44/entities/Clinic.jsonc` and `base44/entities/ContactLog.jsonc`.
  Maps almost 1:1 to two Supabase tables (Clinic + ContactLog, one-to-many via clinic_id).
  Note `providers` is an embedded array of {name, response} — same array-column pattern
  as AP MED's specialty[]/can_help_with[].

- **Tavily search** — `base44/functions/tavilySearch/entry.ts`. ~30-line Deno handler,
  becomes a Next.js route handler (`src/app/api/search/route.ts`) nearly line-for-line.
  Regex phone extraction included.

- **Geocoding** — `base44/functions/geocodeAddress/entry.ts`. Uses the FREE US Census
  geocoder (no API key, no billing). Keep this instead of Google Places.

- **Map** — `src/components/canishadow/MapView.jsx`. react-leaflet + free CARTO tiles.
  Portable to Next.js basically unchanged.

- **Design system** — `src/lib/status.js`. Full "ledger" color system + STATUS_META +
  the effectiveStatus() rule (pin goes green if ANY provider said yes). Zero base44
  dependency — copy as-is.

- **Custom UI** — everything under `src/components/canishadow/` (ClinicDrawer, LogCallForm,
  SearchLogForm, ProviderList, ContactHistory, FilterBar, Legend, Header, StatusBadge).

## What is base44-specific (swap out)

- `src/api/base44Client.js` and all `base44.entities.*` / `base44.functions.invoke(...)`
  calls → replace with Supabase client + Next.js API routes.
- `base44/config.jsonc`, `src/lib/app-params.js`, `src/lib/AuthContext.jsx` (base44 auth)
  → not needed; use Supabase.
